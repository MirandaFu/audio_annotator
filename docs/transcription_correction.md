# 转写纠错机制设计

## 一、问题分析

Whisper 中文转写的典型错误类型：

| 错误类型 | 示例 | 原因 |
|---|---|---|
| 同音字 | "开会" → "开会"（正确）/ "开会" → "开回" | 中文同音字多 |
| 专业术语 | "API" → "A P I" / "K8s" → "K 八 S" | 训练语料中少 |
| 人名/地名 | "张三" → "张三" / "张三丰" → "张三风" | 低频词 |
| 语气词 | "嗯"、"啊"、"就是" |  filler words |
| 数字 | "2024年" → "二〇二四年" / "3.14" → "三点一四" | 数字表达多样 |
| 英文混用 | "OK" → "哦开" | 中英混合 |

## 二、纠错机制设计

### 2.1 用户词典（优先级最高）

允许用户维护一个自定义词典，转写时优先匹配。

```json
{
  "custom_terms": [
    { "term": "数智办公", "hint": "项目名称" },
    { "term": "K8s", "hint": "Kubernetes" },
    { "term": "API", "hint": "接口" },
    { "term": "张三", "hint": "工程师" }
  ]
}
```

**实现方式**：Whisper 的 `initial_prompt` 参数，在转写前提供上下文提示。

**UI**：
- 侧边栏「用户词典」面板
- 支持从已有转写结果中快速添加词条
- 导入/导出词典文件（JSON）

### 2.2 正则批量替换

对转写结果执行预定义的正则替换规则。

```json
{
  "rules": [
    { "pattern": "\\bA P I\\b", "replacement": "API", "type": "term" },
    { "pattern": "嗯[，。]|啊[，。]", "replacement": "", "type": "filler" },
    { "pattern": "张三风", "replacement": "张三丰", "type": "person" }
  ]
}
```

**UI**：规则管理器，支持添加/启用/禁用规则，实时预览效果。

### 2.3 音近字纠错

基于中文拼音相似度的批量纠错。

```
策略：
1. 对转写结果进行分词
2. 对每个词，查找同音/近音候选
3. 结合上下文（前文、后文）选择最可能的词
4. 标记置信度低的词，高亮提示用户确认
```

**简化实现**：
- 使用 `pypinyin` 库将候选词转为拼音
- 使用编辑距离（Levenshtein）匹配同音词
- 结合词频统计选择最可能的词

**依赖**：`pypinyin`（轻量，~500KB）

### 2.4 置信度标记 + 人工审核

Whisper 支持逐词时间戳和概率输出。

```python
segments, info = model.transcribe(audio, word_timestamps=True)
for seg in segments:
    for word in seg.words:
        print(f"{word.word} (prob: {word.probability:.2f})")
```

**策略**：
- `probability < 0.6` → 高亮标记为「不确定」
- `probability < 0.3` → 强烈标记
- UI 中低置信度词显示为黄色/红色下划线
- 提供「一键跳转到下一个不确定词」功能

### 2.5 对比模型投票

用多个模型/参数组合转写同一音频，取投票结果。

```
模型组合：
- small + beam=5
- medium + beam=5
- small + beam=10

对每个时间片段：
- 3 个结果一致 → 高置信度
- 2 个一致 → 中置信度
- 3 个都不同 → 低置信度，人工确认
```

**权衡**：计算量翻倍，但准确率提升明显。可作为可选功能。

### 2.6 上下文一致性检查

检查转写结果中的逻辑一致性。

```
检查项：
1. 数字一致性：同一数字在不同片段中的写法是否统一
   - "3.14" vs "三点一四" → 提示统一
2. 人名一致性：同一人名的写法是否统一
   - "张三" vs "张三" → 检测到一致
   - "张三" vs "张三丰" → 提示确认是否为同一人
3. 时间一致性：提到的日期/时间是否合理
   - "明天开会" → 需要知道当前日期
```

### 2.7 播放复核

提供「点击转写文本 → 播放对应音频」的功能。

**UI 设计**：
- 转写文本区域，每段可点击
- 点击后播放对应音频片段
- 播放时当前文本高亮
- 支持直接编辑文本
- 编辑后自动保存

## 三、推荐实现优先级

### Phase 1：快速见效（1-2 天）

1. **初始提示词优化** — 在转写时传入 `initial_prompt`，提供会议场景上下文
2. **用户词典** — 最简单的干预手段，效果最直接
3. **置信度标记** — 利用 Whisper 自带概率，标记不确定内容

### Phase 2：批量纠错（2-3 天）

4. **正则替换规则** — 处理常见的系统性错误
5. **语气词过滤** — 批量清除"嗯"、"啊"等 filler words

### Phase 3：智能纠错（3-5 天）

6. **音近字纠错** — 拼音相似度匹配
7. **对比模型** — 多模型投票
8. **上下文一致性** — 数字/人名统一性检查

## 五、准确率提升路径

```
当前（small 模型，无优化）
    │
    ▼
+ initial_prompt（场景提示）
    │  预期提升: 5-10%
    ▼
+ 用户词典（领域术语）
    │  预期提升: 3-8%
    ▼
+ 正则规则（系统性错误）
    │  预期提升: 2-5%
    ▼
+ 模型升级（medium）
    │  预期提升: 10-15%（但速度慢 3x）
    ▼
+ 音近字纠错 + 上下文一致性
    │  预期提升: 3-7%
    ▼
目标准确率: 90-95%（专业场景）/ 95-98%（通用场景）
```
